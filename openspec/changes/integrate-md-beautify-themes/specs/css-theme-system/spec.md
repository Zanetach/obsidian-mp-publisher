## 新增需求

### 需求:预设主题系统
系统必须提供 13 个预设 CSS 主题供用户选择，包括 basic、codeGithub、academicPaper、auroraGlass、bauhaus、cyberpunkNeon、knowledgeBase、luxuryGold、morandiForest、neoBrutalism、receipt、sunsetFilm、template。

#### 场景:选择预设主题
- **当** 用户在设置中选择一个预设主题
- **那么** 预览视图必须立即应用该主题的 CSS 样式

### 需求:自定义主题创建
系统必须允许用户创建自定义主题，用户必须能够输入主题名称和 CSS 内容。

#### 场景:创建自定义主题
- **当** 用户点击"新建主题"按钮并输入主题名称和 CSS
- **那么** 系统必须保存该自定义主题到用户设置中

### 需求:自定义主题编辑
系统必须允许用户编辑已有自定义主题的 CSS 内容。

#### 场景:编辑自定义主题
- **当** 用户选择一个自定义主题并点击编辑
- **那么** 系统必须显示主题 CSS 编辑器供用户修改

### 需求:自定义主题删除
系统必须允许用户删除自定义主题，但必须显示确认对话框。

#### 场景:删除自定义主题
- **当** 用户选择删除一个自定义主题并确认
- **那么** 系统必须从用户设置中移除该主题

### 需求:主题导出
系统必须支持将自定义主题导出为 CSS 文件供用户下载。

#### 场景:导出主题
- **当** 用户点击导出按钮
- **那么** 浏览器必须下载包含主题 CSS 的 .css 文件

### 需求:主题导入
系统必须支持用户从外部导入主题 CSS 文件。

#### 场景:导入主题
- **当** 用户选择导入一个 CSS 文件
- **那么** 系统必须解析文件内容并创建新的自定义主题
