## 新增需求

### 需求:主题列表展示
系统必须在设置界面中显示所有可用主题的列表，包括预设主题和自定义主题。

#### 场景:显示主题列表
- **当** 用户打开主题设置页面
- **那么** 必须显示所有预设主题和自定义主题

### 需求:主题预览
用户在选择主题时必须能够预览该主题的效果。

#### 场景:实时主题预览
- **当** 用户悬停在某个主题上或选中该主题
- **那么** 预览视图必须立即显示该主题的效果

### 需求:主题选择保存
系统必须记住用户选择的主题，并在下次打开时自动应用。

#### 场景:保存主题选择
- **当** 用户选择一个新的主题
- **那么** 系统必须将选择保存到用户设置中

### 需求:主题管理器入口
系统必须在设置界面提供主题管理器的入口按钮。

#### 场景:打开主题管理器
- **当** 用户点击"管理主题"按钮
- **那么** 系统必须打开主题管理器模态框

### 需求:主题 CSS 编辑器
系统必须在主题管理器中提供 CSS 编辑器，支持语法高亮。

#### 场景:编辑主题 CSS
- **当** 用户在主题管理器中选择编辑自定义主题
- **那么** 系统必须显示带有语法高亮的 CSS 编辑器
