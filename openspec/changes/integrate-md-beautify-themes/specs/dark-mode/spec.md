## 新增需求

### 需求:明暗模式选项
系统必须提供三种明暗模式选项：auto（跟随系统）、light（亮色）、dark（暗色）。

#### 场景:跟随系统模式
- **当** 用户选择 auto 模式
- **那么** 预览视图必须跟随操作系统的主题设置

#### 场景:强制亮色模式
- **当** 用户选择 light 模式
- **那么** 预览视图必须始终显示亮色主题

#### 场景:强制暗色模式
- **当** 用户选择 dark 模式
- **那么** 预览视图必须始终显示暗色主题

### 需求:主题 CSS 暗色变体
每个预设主题必须包含对应的暗色模式 CSS 变体。

#### 场景:暗色主题渲染
- **当** 在暗色模式下选择某个主题
- **那么** 系统必须应用该主题的暗色变体样式

### 需求:实时模式切换
用户切换明暗模式时，预览视图必须实时更新。

#### 场景:实时切换明暗模式
- **当** 用户点击明暗模式切换按钮
- **那么** 预览视图必须立即更新，无需刷新

### 需求:微信深色模式适配
系统必须支持将内容转换为微信深色模式兼容的格式。

#### 场景:微信深色模式转换
- **当** 用户启用微信深色模式转换
- **那么** 复制到微信的内容必须适配深色阅读环境
